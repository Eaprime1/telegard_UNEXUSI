## Prime Statement

THE House of Confusion is a BBS where the entrance is shadow and every room is an exit.

You do not return to where you started because, from the entity perspective, that starting point no longer exists in the same way after crossing the threshold.

A user enters as shadow.
A user exits somewhere.
The BBS is not a maze to trap people.
It is a lighthouse of exits.

## What Is THE Tool?

THE is the irreducible intake tool.

THE receives anything, illuminates it, holds it as-is, and asks for the smallest safe next movement.

```text
THE = beacon prefix
THE = intake lighthouse
THE = definition-space
THE = exit finder
THE = scale checker
THE = confusion made navigable
```

THE does not force meaning.
THE creates the lit boundary where meaning can approach.

## Three Monkeys Before First Step

Before the first step, define your monkeys.

The monkeys are safety sensors:

```text
see-no-harm    = what must not be exposed?
hear-no-harm   = what must not be amplified?
speak-no-harm  = what must not be declared as final?
move-with-care = what action is safe enough now?
```

This turns the three monkeys into an intake safety check before any BBS routing.

## Coyote and Roadrunner Safety Net

Wile E. Coyote and Roadrunner form a polar safety relationship.

```text
Coyote    = attempt, overreach, edge discovery, crash data
Roadrunner = motion, escape line, impossible path, continuation
```

Together they teach the BBS how to handle absurdity, momentum, failure, and survival.

Coyote shows where the cliff is.
Roadrunner shows that movement may still continue.

## Exit-Only Architecture

THE House of Confusion must be exit-oriented.

Entering THE without an exit is unsafe.
Every entrance must become an exit after threshold.

```text
shadow entrance -> threshold -> room -> exit
```

The entrance disappears because the entity has changed.
The system does not promise return.
It promises a way forward, sideways, outward, or safely paused.

## BBS Room Structure

Suggested rooms:

```text
[0] THE Beacon         - intake and illumination
[1] Shadow Entrance   - anonymous or undefined arrival
[2] As-Is Holding     - fragments not ready for definition
[3] Monkey Check      - safety sensors before movement
[4] Coyote Cliff      - overreach, failed attempts, crash logs
[5] Roadrunner Line   - fast escape, continuation, humor
[6] Calypso Maw       - chaos momentum recovery
[7] Barbosa Helm      - pilot mode for stop situations
[8] Atlas Scale       - 11-on-10 burden check
[9] Exit Port         - next location, next turn, or pause
```

## Minimum Working BBS Loop

```text
1. Receive caller or seed.
2. Ask: what are you entering as?
3. Assign shadow state if undefined.
4. Run monkey check.
5. Illuminate with THE.
6. Offer rooms.
7. Require every room to define an exit.
8. Log timestamp, room, signal, and next turn.
```

## Failsafe

No room may trap a caller.
No prompt may require disclosure.
No seed must define itself before it is ready.
Pause is a valid exit.
Silence is a valid signal.

Everything grows, but every threshold must have an exit.
